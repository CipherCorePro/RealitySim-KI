types.ts

Diese Datei definiert die Kern-TypeScript-Schnittstellen und -Typen, die die Datenstrukturen der Simulationswelt und ihrer Entitäten repräsentieren.

Interface: Beliefs

interface Beliefs { [key: string]: number; }

Repräsentiert ein Netzwerk von Überzeugungen, wobei jeder Schlüssel eine Überzeugung und der Wert ihre Stärke (numerisch) darstellt.

Interface: Emotions

interface Emotions { [key: string]: number; }

Repräsentiert die emotionalen Zustände, wobei jeder Schlüssel eine Emotion und der Wert ihre Intensität darstellt.

Typ: RelationshipType

type RelationshipType = 'stranger' | 'acquaintance' | 'friend' | 'rival' | 'partner' | 'spouse' | 'ex-partner';

Definiert die möglichen Typen von Beziehungen zwischen Agenten.

Interface: Relationship

interface Relationship { type: RelationshipType; score: number; disposition: Emotions; }

Beschreibt eine Beziehung zwischen zwei Agenten.

Eigenschaften:

  • type: RelationshipType - Der Typ der Beziehung (z.B. 'friend', 'rival').
  • score: number - Ein numerischer Wert, der die Stärke oder Qualität der Beziehung angibt.
  • disposition: Emotions - Langfristig erlernte Emotionen gegenüber diesem Agenten.

Interface: Resonance

interface Resonance { [key: string]: number; }

Repräsentiert die Resonanz oder Präferenz eines Agenten für bestimmte Aktionen oder Konzepte.

Interface: LogEntry

interface LogEntry { key: string; params?: { [key: string]: any }; }

Definiert einen Eintrag im Ereignisprotokoll der Simulation.

Eigenschaften:

  • key: string - Der Schlüssel für den Protokolleintrag (oft für Übersetzungen verwendet).
  • params?: { [key: string]: any } - Optionale Parameter, die im Protokolleintrag ersetzt werden können.

Interface: SocialMemoryEntry

interface SocialMemoryEntry { agentId: string; action: string; result: 'accepted' | 'rejected' | 'initiated' | 'reciprocated'; emotionalImpact: number; timestamp: number; }

Repräsentiert einen Eintrag im sozialen Gedächtnis eines Agenten, der vergangene Interaktionen festhält.

Eigenschaften:

  • agentId: string - Die ID des interagierenden Agenten.
  • action: string - Die durchgeführte Aktion.
  • result: 'accepted' | 'rejected' | 'initiated' | 'reciprocated' - Das Ergebnis der Interaktion.
  • emotionalImpact: number - Der emotionale Einfluss der Interaktion (-1 bis 1).
  • timestamp: number - Der Zeitpunkt der Interaktion.

Interface: ActionExecutionResult

interface ActionExecutionResult { newEnvironment: EnvironmentState; log: LogEntry; sideEffects?: { createAgent?: Partial<Agent> & { name: string, description: string, parents: [Agent, Agent] }; updateReligion?: { agentId: string, newReligionId: string | null }; createEntity?: Partial<Entity>; }; }

Definiert das Ergebnis der Ausführung einer Aktion durch einen Agenten.

Eigenschaften:

  • newEnvironment: EnvironmentState - Der neue Zustand der Umgebung nach der Aktion.
  • log: LogEntry - Ein Protokolleintrag, der die Aktion beschreibt.
  • sideEffects?: object - Optionale Nebenwirkungen der Aktion, wie das Erstellen neuer Agenten oder Entitäten.

Interface: Action

interface Action { name: string; description: string; beliefKey?: string; execute?: (agent: Agent, allAgents: Map<string, Agent>, allEntities: Map<string, Entity>, environment: EnvironmentState) => ActionExecutionResult; }

Definiert eine Aktion, die Agenten ausführen können.

Hinweis: Das execute-Feld ist mit @ts-ignore markiert, was auf eine potenzielle zirkuläre Abhängigkeit oder eine komplexe Typinferenz hinweisen könnte, die manuell ignoriert wird.

Eigenschaften:

  • name: string - Der Name der Aktion.
  • description: string - Eine Beschreibung der Aktion.
  • beliefKey?: string - Ein optionaler Schlüssel, der die Aktion mit einer bestimmten Überzeugung verknüpft.
  • execute?: (agent: Agent, allAgents: Map<string, Agent>, allEntities: Map<string, Entity>, environment: EnvironmentState) => ActionExecutionResult - Eine optionale Funktion, die die Logik der Aktion implementiert.

Typ: ResourceType

type ResourceType = 'food' | 'water' | 'wood' | 'medicine';

Definiert die möglichen Typen von Ressourcen in der Simulation.

Interface: Inventory

interface Inventory { [resourceType: string]: number; }

Repräsentiert das Inventar eines Agenten, das die Mengen verschiedener Ressourcen speichert.

Interface: Agent

interface Agent { id: string; name: string; description: string; x: number; y: number; beliefNetwork: Beliefs; emotions: Emotions; resonance: Resonance; socialMemory: SocialMemoryEntry[]; lastActions: Action[]; adminAgent: boolean; health: number; isAlive: boolean; sickness?: string | null; conversationHistory: { speakerName: string; message: string; }[]; age: number; genome: string[]; relationships: { [agentId: string]: Relationship }; cultureId: string | null; religionId: string | null; role: string | null; offspringCount: number; hunger: number; thirst: number; fatigue: number; inventory: Inventory; }

Definiert die Eigenschaften eines KI-Agenten in der Simulation.

Eigenschaften:

  • id: string - Eindeutige ID des Agenten.
  • name: string - Name des Agenten.
  • description: string - Beschreibung des Agenten.
  • x: number - X-Koordinate der Position des Agenten.
  • y: number - Y-Koordinate der Position des Agenten.
  • beliefNetwork: Beliefs - Das Überzeugungsnetzwerk des Agenten.
  • emotions: Emotions - Die aktuellen Emotionen des Agenten.
  • resonance: Resonance - Die Resonanz des Agenten für Aktionen.
  • socialMemory: SocialMemoryEntry[] - Das soziale Gedächtnis des Agenten.
  • lastActions: Action[] - Die zuletzt ausgeführten Aktionen des Agenten.
  • adminAgent: boolean - Gibt an, ob der Agent ein Administrator-Agent ist.
  • health: number - Der Gesundheitszustand des Agenten.
  • isAlive: boolean - Gibt an, ob der Agent am Leben ist.
  • sickness?: string | null - Aktuelle Krankheit des Agenten, falls vorhanden.
  • conversationHistory: { speakerName: string; message: string; }[] - Der Konversationsverlauf des Agenten.
  • age: number - Das Alter des Agenten.
  • genome: string[] - Das Genom des Agenten.
  • relationships: { [agentId: string]: Relationship } - Beziehungen zu anderen Agenten.
  • cultureId: string | null - Die ID der Kultur, der der Agent angehört.
  • religionId: string | null - Die ID der Religion, der der Agent angehört.
  • role: string | null - Die Rolle des Agenten (z.B. 'Leader', 'Healer').
  • offspringCount: number - Anzahl der Nachkommen des Agenten.
  • hunger: number - Hungerlevel (0-100, 100 = verhungert).
  • thirst: number - Durstlevel (0-100, 100 = dehydriert).
  • fatigue: number - Müdigkeitslevel (0-100, 100 = erschöpft).
  • inventory: Inventory - Das Inventar des Agenten.

interface Entity { id: string; name: string; description: string; x: number; y: number; isResource?: boolean; resourceType?: ResourceType; quantity?: number; }

Definiert die Eigenschaften einer Entität (Objekt) in der Simulationswelt.

Eigenschaften:
