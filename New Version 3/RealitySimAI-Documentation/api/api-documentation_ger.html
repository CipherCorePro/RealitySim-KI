Technische Dokumentation: Ver. 4 RealitySim AI

Projektübersicht

Name: Ver. 4 RealitySim AI

Beschreibung: Eine interaktive webbasierte Simulationsumgebung, in der KI-Agenten mit einzigartigen Überzeugungen und Erinnerungen mit einer dynamischen Welt interagieren. Steuern Sie Agenten mit natürlicher Sprache, führen Sie Simulationsschritte aus und visualisieren Sie das emergente kognitive Verhalten des Systems.

Dateien

Datei: metadata.json

metadata.json

Diese Datei enthält Metadaten über das Projekt, wie den Namen und eine kurze Beschreibung.

Struktur

{
  "name": "Ver. 4 RealitySim AI",
  "description": "An interactive web-based simulation environment where AI agents with unique beliefs and memories interact with a dynamic world. Control agents with natural language, run simulation steps, and visualize the emergent cognitive behavior of the system.",
  "requestFramePermissions": [],
  "prompt": ""
}

Felder

  • name: string - Der Name des Projekts.
  • description: string - Eine kurze Beschreibung des Projekts.
  • requestFramePermissions: Array<any> - Eine Liste von Berechtigungen, die möglicherweise für Iframes angefordert werden. (Derzeit leer)
  • prompt: string - Ein Feld für einen Start-Prompt. (Derzeit leer)

Datei: types.ts

types.ts

Diese Datei definiert alle zentralen TypeScript-Schnittstellen und Typen, die die Datenstrukturen für die Simulation, Agenten, Entitäten und den Weltzustand beschreiben.

Schnittstellen und Typen

Beliefs

interface Beliefs { [key: string]: number; }
Repräsentiert die Überzeugungen eines Agenten als Schlüssel-Wert-Paare, wobei der Wert die Intensität der Überzeugung angibt.

Emotions

interface Emotions { [key: string]: number; }
Repräsentiert die Emotionen eines Agenten als Schlüssel-Wert-Paare, wobei der Wert die Intensität der Emotion angibt.

Personality

interface Personality { openness: number; conscientiousness: number; extraversion: number; agreeableness: number; neuroticism: number; }
Definiert die Persönlichkeitsmerkmale eines Agenten basierend auf dem Big Five-Modell. Werte liegen zwischen 0 und 1.
Parameter:
  • openness: number - Offenheit für Erfahrungen.
  • conscientiousness: number - Gewissenhaftigkeit.
  • extraversion: number - Extraversion.
  • agreeableness: number - Verträglichkeit.
  • neuroticism: number - Neurotizismus.

Skills

interface Skills { [key: string]: number; }
Repräsentiert die Fähigkeiten eines Agenten als Schlüssel-Wert-Paare, z.B. { healing: 15, woodcutting: 22, rhetoric: 5 }.

GoalType

type GoalType = 'becomeLeader' | 'buildLargeHouse' | 'masterSkill' | 'avengeRival' | 'achieveWealth' | 'mentorYoungAgent' | 'seekCounseling';
Definiert die möglichen Typen von Zielen, die ein Agent haben kann.

Goal

interface Goal { type: GoalType; status: 'active' | 'completed' | 'failed'; progress: number; description: string; targetId?: string; }
Beschreibt ein Ziel, das ein Agent verfolgt.
Parameter:
  • type: GoalType - Der Typ des Ziels.
  • status: 'active' | 'completed' | 'failed' - Der aktuelle Status des Ziels.
  • progress: number - Der Fortschritt des Ziels (0-100).
  • description: string - Eine Beschreibung des Ziels.
  • targetId?: string - Optional: Die ID des Ziel-Agenten oder der Ziel-Entität.

Trauma

interface Trauma { event: string; timestamp: number; intensity: number; }
Repräsentiert ein traumatisches Ereignis, das ein Agent erlebt hat.
Parameter:
  • event: string - Beschreibung des traumatischen Ereignisses.
  • timestamp: number - Zeitpunkt des Ereignisses.
  • intensity: number - Intensität des Traumas (0-1).

RelationshipType

type RelationshipType = 'stranger' | 'acquaintance' | 'friend' | 'rival' | 'partner' | 'spouse' | 'ex-partner';
Definiert die möglichen Typen von Beziehungen zwischen Agenten.

Relationship

interface Relationship { type: RelationshipType; score: number; disposition: Emotions; }
Beschreibt die Beziehung eines Agenten zu einem anderen.
Parameter:
  • type: RelationshipType - Der Typ der Beziehung.
  • score: number - Ein numerischer Wert, der die Stärke oder Qualität der Beziehung angibt.
  • disposition: Emotions - Emotionale Disposition gegenüber dem anderen Agenten.

Resonance

interface Resonance { [key:string]: number; }
Repräsentiert die Resonanz eines Agenten mit bestimmten Konzepten oder Ideen.

LogEntry

interface LogEntry { key: string; params?: { [key: string]: any }; }
Ein Eintrag für das Simulationsprotokoll.
Parameter:
  • key: string - Ein Schlüssel zur Identifizierung des Protokolleintrags, oft für Übersetzungen.
  • params?: { [key: string]: any } - Optionale Parameter, die für die Formatierung des Protokolleintrags verwendet werden.

SocialMemoryEntry

interface SocialMemoryEntry { agentId: string; action: string; result: 'accepted' | 'rejected' | 'initiated' | 'reciprocated' | 'observed'; emotionalImpact: number; timestamp: number; info?: string; }
Ein Eintrag im sozialen Gedächtnis eines Agenten, der Interaktionen mit anderen Agenten festhält.
Parameter:
  • agentId: string - Die ID des beteiligten Agenten.
  • action: string - Die durchgeführte Aktion.
  • result: 'accepted' | 'rejected' | 'initiated' | 'reciprocated' | 'observed' - Das Ergebnis der Aktion.
  • emotionalImpact: number - Der emotionale Einfluss der Aktion auf den Agenten (-1 bis 1).
  • timestamp: number - Der Zeitpunkt des Ereignisses.
  • info?: string - Optionale zusätzliche Informationen.

ResourceType

type ResourceType = 'food' | 'water' | 'wood' | 'medicine' | 'iron';
Definiert die Typen von Ressourcen, die in der Welt existieren können.

CraftedItemType

type CraftedItemType = 'sword' | 'plow' | 'advanced_medicine' | 'iron_ingot';
Definiert die Typen von hergestellten Gegenständen.

ItemType

type ItemType = ResourceType | CraftedItemType;
Ein Union-Typ, der alle möglichen Gegenstandstypen (Ressourcen und hergestellte Gegenstände) umfasst.

Inventory

interface Inventory { [item: string]: number; }
Repräsentiert das Inventar eines Agenten oder einer Entität als Schlüssel-Wert-Paare (Gegenstand: Menge).

Recipe

interface Recipe { name: string; output: { item: CraftedItemType, quantity: number }; ingredients: { [key in ItemType]?: number }; requiredSkill?: { skill: keyof Skills, level: number }; requiredTech?: string; }
Beschreibt ein Rezept zur Herstellung von Gegenständen.
Parameter:
  • name: string - Name des Rezepts.
  • output: { item: CraftedItemType, quantity: number } - Der Gegenstand und die Menge, die hergestellt werden.
  • ingredients: { [key in ItemType]?: number } - Benötigte Zutaten und deren Mengen.
  • requiredSkill?: { skill: keyof Skills, level: number } - Optional: Benötigte Fähigkeit und Level.
  • requiredTech?: string - Optional: Benötigte Technologie.

TradeOffer

interface TradeOffer { offerId: string; fromAgentId: string; item: ItemType; quantity: number; price: number; }
Repräsentiert ein Handelsangebot auf einem Markt.
Parameter:
  • offerId: string - Eindeutige ID des Angebots.
  • fromAgentId: string - ID des Agenten, der das Angebot macht.
  • item: ItemType - Der angebotene Gegenstand.
  • quantity: number - Die Menge des Gegenstands.
  • price: number - Der Preis pro Einheit.

Market

interface Market { id: string; name: string; listings: TradeOffer[]; }
Repräsentiert einen Markt in der Welt.
Parameter:
  • id: string - Eindeutige ID des Marktes.
  • name: string - Name des Marktes.
  • listings: TradeOffer[] - Aktuelle Handelsangebote auf diesem Markt.

Law

interface Law { id: string; name: string; description: string; violatingAction: string; punishment: { type: 'fine' | 'arrest'; amount: number; }; }
Definiert ein Gesetz in der Simulation.
Parameter:
  • id: string - Eindeutige ID des Gesetzes.
  • name: string - Name des Gesetzes.
  • description: string - Beschreibung des Gesetzes.
  • violatingAction: string - Der Name der Aktion, die dieses Gesetz verbietet.
  • punishment: { type: 'fine' | 'arrest'; amount: number; } - Die Strafe für die Verletzung des Gesetzes (Geldstrafe oder Verhaftungsdauer).

Government

interface Government { type: 'monarchy' | 'democracy'; leaderId: string | null; laws: Law[]; }
Repräsentiert die Regierung in der Welt.
Parameter:
  • type: 'monarchy' | 'democracy' - Der Typ der Regierungsform.
  • leaderId: string | null - Die ID des Anführers der Regierung (falls vorhanden).
  • laws: Law[] - Eine Liste der geltenden Gesetze.

Election

interface Election { isActive: boolean; candidates: string[]; votes: { [candidateId: string]: number }; termEndDate: number; }
Beschreibt den Zustand einer Wahl.
Parameter:
  • isActive: boolean - Gibt an, ob eine Wahl aktiv ist.
  • candidates: string[] - Eine Liste der IDs der Kandidaten.
  • votes: { [candidateId: string]: number } - Die Stimmen für jeden Kandidaten.
  • termEndDate: number - Das Enddatum der Amtszeit des gewählten Anführers.

Technology

interface Technology { id: string; name: string; description: string; researchCost: number; unlocks: { actions?: string[]; recipes?: string[]; }; requiredTech?: string[]; }
Definiert eine Technologie, die erforscht werden kann.
Parameter:
  • id: string - Eindeutige ID der Technologie.
  • name: string - Name der Technologie.
  • description: string - Beschreibung der Technologie.
  • researchCost: number - Die Kosten für die Erforschung der Technologie.
  • unlocks: { actions?: string[]; recipes?: string[]; } - Was diese Technologie freischaltet (Aktionen, Rezepte).
  • requiredTech?: string[] - Optional: IDs von Technologien, die vor dieser erforscht werden müssen.

Agent

interface Agent { id: string; name: string; description: string; x: number; y: number; beliefNetwork: Beliefs; emotions: Emotions; resonance: Resonance; socialMemory: SocialMemoryEntry[]; lastActions: Action[]; adminAgent: boolean; health: number; isAlive: boolean; sickness?: string | null; conversationHistory: { speakerName: string; message: string; }[]; age: number; genome: string[]; relationships: { [agentId: string]: Relationship }; cultureId: string | null; religionId: string | null; role: string | null; offspringCount: number; hunger: number; thirst: number; fatigue: number; inventory: Inventory; personality: Personality; goals: Goal[]; stress: number; socialStatus: number; skills: Skills; trauma: Trauma[]; unconsciousState?: { [key: string]: number }; currency: number; imprisonedUntil?: number; }
Repräsentiert einen KI-Agenten in der Simulation mit all seinen Eigenschaften und Zuständen.
Parameter:
  • id: string - Eindeutige ID des Agenten.
  • name: string - Name des Agenten.
  • description: string - Beschreibung des Agenten.
  • x: number - X-Koordinate der Position des Agenten.
  • y: number - Y-Koordinate der Position des Agenten.
  • beliefNetwork: Beliefs - Das Überzeugungsnetzwerk des Agenten.
  • emotions: Emotions - Die aktuellen Emotionen des Agenten.
  • resonance: Resonance - Die Resonanz des Agenten.
  • socialMemory: SocialMemoryEntry[] - Das soziale Gedächtnis des Agenten.
  • lastActions: Action[] - Die letzten Aktionen des Agenten.
  • adminAgent: boolean - Gibt an, ob der Agent ein Administrator-Agent ist.
  • health: number - Die Gesundheit des Agenten.
  • isAlive: boolean - Gibt an, ob der Agent am Leben ist.
  • sickness?: string | null - Optional: Die Krankheit, an der der Agent leidet.
  • conversationHistory: { speakerName: string; message: string; }[] - Der Konversationsverlauf des Agenten.
  • age: number - Das Alter des Agenten.
  • genome: string[] - Das Genom des Agenten.
  • relationships: { [agentId: string]: Relationship } - Beziehungen zu anderen Agenten.
  • cultureId: string | null - Die ID der Kultur, der der Agent angehört.
  • religionId: string | null - Die ID der Religion, der der Agent angehört.
  • role: string | null - Die Rolle des Agenten.
  • offspringCount: number - Die Anzahl der Nachkommen des Agenten.
  • hunger: number - Das Hungerlevel des Agenten.
  • thirst: number - Das Durstlevel des Agenten.
  • fatigue: number - Das Müdigkeitslevel des Agenten.
  • inventory: Inventory - Das Inventar des Agenten.
  • personality: Personality - Die Persönlichkeit des Agenten.
  • goals: Goal[] - Die Ziele des Agenten.
  • stress: number - Das Stresslevel des Agenten.
  • socialStatus: number - Der soziale Status des Agenten.
  • skills: Skills - Die Fähigkeiten des Agenten.
  • trauma: Trauma[] - Die Traumata des Agenten.
  • unconsciousState?: { [key: string]: number } - Optional: Unbewusste Zustände oder Modifikatoren.
  • currency: number - Die Währung des Agenten.
  • imprisonedUntil?: number - Optional: Der Zeitpunkt, bis zu dem der Agent inhaftiert ist.

Entity

interface Entity { id: string; name: string; description: string; x: number; y: number; isResource?: boolean; resourceType?: ResourceType; quantity?: number; ownerId?: string | null; isMarketplace?: boolean; isJail?: boolean; inmates?: string[]; }
Repräsentiert eine Entität in der Welt (z.B. Gebäude, Ressourcenpunkte).
Parameter:
  • id: string - Eindeutige ID der Entität.
  • name: string - Name der Entität.
  • description: string - Beschreibung der Entität.
  • x: number - X-Koordinate der Position der Entität.
  • y: number - Y-Koordinate der Position der Entität.
  • isResource?: boolean - Optional: Gibt an, ob die Entität eine Ressource ist.
  • resourceType?: ResourceType - Optional: Der Typ der Ressource, falls zutreffend.
  • quantity?: number - Optional: Die Menge der Ressource, falls zutreffend.
  • ownerId?: string | null - Optional: Die ID des Besitzers der Entität.
  • isMarketplace?: boolean - Optional: Gibt an, ob die Entität ein Marktplatz ist.
  • isJail?: boolean - Optional: Gibt an, ob die Entität ein Gefängnis ist.
  • inmates?: string[] - Optional: Eine Liste der IDs der Insassen, falls es ein Gefängnis ist.

EnvironmentState

interface EnvironmentState { [key: string]: any; width: number; height: number; time: number; election: Election | null; }
Repräsentiert den Zustand der Umgebung.
Parameter:
  • width: number - Die Breite der Umgebung.
  • height: number - Die Höhe der Umgebung.
  • time: number - Die aktuelle Simulationszeit.
  • election: Election | null - Der Zustand der Wahl, falls eine aktiv ist.
